JAVA 스트림의 핵심: 스트림 파이프라인.

스트림? 마치 공장의 컨베이어 벨트와 같음. 

과정:
1.재료 올리기 -> 2.가공하기 -> 3.포장하기 (해당 순서를 엄격하게 따름)

1. 재료 올리기(생성하기)
스트림을 처음 시작 하는 단계. 컬렉션(List, Set)이나 배열, 파일 등에서 데이터를 가져와 스트림을 제작.

대표적으로...
list.stream() : 리스트에서 스트림 생성, Arrays.stream(arr): 배열에서 생성, Files.walk(path): 파일 경로에서 생성

2. 가공하기(중간연산)
   특징:
   여러번 계속 연결해서 쓸 수 있음. .filter().map().sorted().distinct() ....    각각 필터링, 데이터 변환(A->B로 변환) ,정렬...., 중복 제거
   
   반환값: 또 다른 Stream을 뱉어냄. 그래서 체인처럼 연결된다.

3.포장/출고하기(최종 연산)
  가공된 데이터를 최종적으로 소모하는 단계. 이 함수가 호출되어야만 스트림이 동작한다
  
  특징:
  마지막에 딱 한 번만 쓸 수 있음.
  쓰고 나면 스트림이 closed되어 더 이상 못씁니다.
  
  반환값: Stream이 아닌 진짜 결과값(void List int boolean ..... 등) 이 나옴.
  
  ex) forEach: 하나씩 꺼내서 행동함 (출력 등)

      collect	리스트나 맵으로 묶어서 반환

      count:	개수 세기

      findFirst:	첫 번째 것 가져오기
